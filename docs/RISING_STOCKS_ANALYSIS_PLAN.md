# 당일 주도 테마 및 급등주 정밀 분석 서비스 구축 계획 (v1.0)

## 1. 개요 (Objective)
당일 시장에서 가장 강력하게 상승하는 종목들을 포착하여, **[재료(뉴스/공시) + 차트(80봉) + 과거 히스토리]**를 결합한 입체적인 분석 리포트를 생성하고 이를 지식 베이스화하여 누적합니다.

## 2. 분석 시간 및 워크플로우 (Scheduling)
| 구분 | 시간 | 주요 작업 내용 |
| :--- | :--- | :--- |
| **장중 리포트** | **10:00 AM** | 오전장 주도 테마 포착, 초기 급등 사유 분석, 80봉 차트 위치 진단 |
| **최종 리포트** | **15:40 PM** | 장 마감 데이터 확정, 당일 시황 총평 작성, DB 최종 아카이빙 |

## 3. 데이터 아키텍처 (Data Sources)
1.  **종목 리스트 (Kiwoom API):** 
    *   `ka10027`: 전일대비 등락률 상위 종목 추출
    *   `ka10030`: 거래대금 상위(주도주) 종목 추출 (상승률이 낮아도 거래량이 터진 종목 포함)
2.  **기술적 분석 (Kiwoom API):**
    *   `ka10081`: 최신 80~100봉 일봉 데이터 수집 (추세 및 매물대 분석용)
3.  **상승 사유 (External API):**
    *   **네이버 뉴스 API:** 종목명 키워드 기반 최신 기사 수집
    *   **OpenDART API:** 최근 주요 공시(수주, 실적 등) 조회
4.  **지식 베이스 (Local DB):**
    *   이전 날짜의 분석 리포트를 조회하여 동일 재료 여부 및 연속성 판별

## 5. AI 분석 전략 (Implementation Logic)

### [하이브리드 프롬프트 구성]
1.  **과거 맥락 주입:** DB에서 해당 종목의 과거 급등 기록을 조회하여 AI에게 사전 정보로 제공. ("이 종목은 지난 N월에도 동일 재료로 급등함")
2.  **차트 관점 통합:** 80봉 데이터를 수치화하여 AI에게 전달 (예: "현재 주가가 80일 내 최고가 대비 5% 지점임").
3.  **지속성 판별:** 재료의 성격(단발성 vs 구조적 성장)을 구분하여 점수화(Sustainability Score).

## 6. UI/UX 구상 (User Interface)

### [섹션 1: 일일 총평 리포트]
*   날짜별 헤더에 위치. 
*   "오늘은 반도체 세정 장비 섹터가 시장을 주도했으며, 외인 수급이 특정 테마에 쏠림 현상을 보임..." 등의 텍스트 요약.

### [섹션 2: 주도주 데이터 테이블]
*   **정렬:** 상승률 또는 거래대금 순.
*   **특수 컬럼:** 
    *   `섹터`: 태그 형태로 표시 (예: #반도체 #AI).
    *   `상승 사유`: 클릭 시 상세 뉴스 링크 및 AI 요약 팝업.
    *   `알림`: 과거 기록이 있는 경우 '연속성 감지' 아이콘 표시.

## 7. 기대 효과
1.  **개별 종목 단기 분석 탈피:** 매일 축적되는 데이터를 통해 종목과 테마의 순환매 패턴을 AI가 스스로 학습.
2.  **차트 기반의 안전진단:** 재료가 아무리 좋아도 고점권 매물대의 위험을 AI가 정량적으로 경고.
3.  **지표 중심의 투자 결정:** 개인의 감정이 아닌, 데이터와 누적된 지식 기반의 객관적 투자 판단 보조.

## 8. 분석 지능 고도화 (Self-Learning) 전략

분석 엔진이 단순히 데이터를 조회하는 것을 넘어, 자신의 예측 결과와 실제 시장 데이터를 대조하며 스스로 지능을 높여가는 '자기 진화 루프'를 구축합니다.

### 8.1. 사후 복기 및 피드백 루프 (Retrospective Loop)
*   **복기 시점:** 분석 발생 3일 후 및 7일 후 (장중 16:30 배치 작업).
*   **대조 데이터:** AI가 예측한 '이슈 지속성' 점수 vs 실제 3/7일간의 주가 등락률 및 거래량 추이.
*   **분석 (Why?):** 왜 AI의 예측이 맞았는지 혹은 틀렸는지 AI 스스로 원인을 분석하게 합니다. (예: "기관의 차익실현 물량 과다를 간과함", "테마 대장주 교체 현상을 포착하지 못함")

### 8.2. 동적 스킬스(Skills) 업데이트 및 지식 전이
*   **성공/실패 패턴 아카이브:** 복기 결과를 통해 얻은 교훈을 `ai_learning_log` 테이블과 `.agents/skills/rising_stock_skill.md`에 동적으로 업데이트합니다.
*   **자기 교정형 프롬프트:** 다음 분석 시, 과거의 오답 노트와 성공 사례 패턴을 프롬프트(`systemInstruction`)에 주입하여 동일한 실수를 반복하지 않도록 합니다.

### 8.3. 섹터별/패턴별 적중률(Confidence Score) 관리
*   **정량적 데이터화:** 섹터(반도체, 바이오 등) 및 분석 패턴(눌림목, 돌파 등)별로 AI의 적중률을 산출합니다.
*   **신뢰도 출력:** 리포트 생성 시, "이 종목은 AI의 반도체 섹터 분석 적중률(85%) 기반으로 신뢰도가 높습니다"와 같은 메타 데이터를 함께 제공합니다.

### 8.4. 신류 테이블 추가 설계 (`ai_learning_log`)
| 컬럼명 | 타입 | 설명 |
| :--- | :--- | :--- |
| `original_report_id` | INTEGER | 기존 분석 리포트와의 연결고리 (FK) |
| `prediction_accuracy` | TEXT | SUCCESS(적중) / FAIL(빗나감) / NEUTRAL |
| `actual_performance` | REAL | 예측 이후 N일간의 주가 등락률 |
| `learning_point` | TEXT | 이번 분석을 통해 배운 기술적/재료적 교훈 (지식화) |
| `sector` | TEXT | 분석 대상 섹터 |

## 9. UI/UX 상세 구현 계획

사용자 경험과 개발 효율성을 동시에 고려하여 기존 '즐겨찾기' 레이아웃을 최대한 활용합니다.

### 9.1. 메뉴 및 레이아웃
*   **메뉴명:** '급등주리포트' (Sidebar 아이콘: `TrendingUp`)
*   **레이아웃 구조:** '즐겨찾기' 메뉴와 동일한 2단 분할 레이아웃 적용.
    *   **왼쪽 패널:** 종목 리스트 (날짜별 섹션 그룹화)
    *   **오른쪽 패널:** 상단(차트), 하단(종목 상세 정보 탭)

### 9.2. 왼쪽 종목 리스트 상세
*   **날짜별 섹션 헤더:** 리포트가 생성된 날짜별로 종목을 그룹화하여 표시.
*   **정렬 로직:** 각 날짜 내에서는 최종 리포트 작성 시점의 '주가 상승률'이 높은 순서대로 정렬 (불필요한 실시간 재계산 최소화).
*   **주요 정보 표시:** 종목명, 상승률, 그리고 **'AI 분석 점수'**(향후 추가 상승 여력 지표)를 한눈에 볼 수 있도록 배치.

### 9.3. 오른쪽 상세 정보 및 리포트 탭
*   **'리포트' 탭 신설:** 기존 [DART, 재무, 노트] 탭에 [리포트] 탭을 추가.
*   **기본 탭 설정:** '급등주리포트' 메뉴 진입 시, 하단 정보 영역에서 **'리포트' 탭이 기본(Default)으로 선택**되어 표시되도록 설정.
*   **내용 구성:** AI가 생성한 당일 분석 리포트를 마크다운 형식으로 렌더링.

### 9.4. 개발 효율화 전략
*   `Watchlist.tsx`의 레이아웃 로직을 추상화하거나 재활용하여 일관된 디자인 시스템 유지.
*   `StockChart`, `StockSchedules` (DART), `StockFinancials` 등 이미 검증된 공용 컴포넌트를 그대로 임베딩.

### 9.5. AI 점수별 컬러 배지 적용 원칙
AI 분석 점수(0~100)에 따라 배지의 배경색을 차등 적용하여 직관적인 투자 판단을 돕습니다.
*   **Strong Buy (80~100):** `#22C55E` (Green) - 강력한 재료와 차트 정배열, 높은 지속성.
*   **Buy (60~79):** `#EAB308` (Yellow) - 양호한 재료와 수급, 단기 반등 가능성.
*   **Hold/Watch (40~59):** `#F97316` (Orange) - 재료는 있으나 차트 위치 부담 또는 수급 분산.
*   **Caution (0~39):** `#EF4444` (Red) - 단발성 재료 또는 고점권 차익실현 위험군.

## 10. 프롬프트 설계 (Prompt Engineering) 상세

AI가 데이터를 단순 요약하는 수준을 넘어, 전문 투자자 수준의 통찰을 도출하도록 프롬프트를 정교하게 설계합니다.

### 10.1. 시스템 인스트럭션 (System Instructions)
*   **Role:** 15년 경력의 대한민국 시장 전문 주식 애널리스트 및 퀀트 전문가.
*   **Tone:** 객관적이고 데이터 중심적이며, 지나친 낙관이나 비관을 배제함.
*   **Knowledge Base:** `.agents/skills/kiwoom_app_development/rising_stock_skill.md`의 내용을 기반 지식으로 활용.
*   **Mission:** 제공된 뉴스 컨텍스트와 80봉 일봉 데이터를 결합하여 상승의 '진성성'을 판별하고, 3~5일 내 추가 상승 가능성을 점수화(0-100)함.

### 10.2. 입력 데이터 구조 (Few-Shot & Context)
AI에게 전달될 프롬프트는 다음과 같은 구조를 가집니다.
```json
{
  "stock": { "name": "종목명", "code": "코덱", "change": "+25.4%" },
  "technical": {
    "chart_80_summary": "이평선 정배열 여부, 현재 주가의 80일 내 위치(%), 대량 거래 발생 구간 등",
    "support_resistance": "주요 매물대 위치 및 돌파 여부"
  },
  "fundamental": {
    "news": ["뉴스 제목 1...", "뉴스 제목 2..."],
    "disclosures": ["최신 공시명 1...", "최신 공시명 2..."]
  },
  "past_log": "이 종목의 과거 유효 분석 기록 (있을 경우)"
}
```

### 10.3. 출력 형식 (JSON Response)
UI 및 DB 연동을 위해 반드시 정해진 JSON 형식을 따르도록 강제합니다. (분석 결과물은 UI에서 Markdown으로 변환하여 출력)

## 11. 설정 메뉴 확장 및 Naver API 연동 계획

### 11.1. Naver 뉴스 API 설정 UI
*   `Settings.tsx`의 '외부 API' 탭에 'Naver Open API' 섹션 추가.
*   **입력 필드:** `Client ID`, `Client Secret` 저장 기능.
*   **테스트 버튼:** "연결 테스트" 클릭 시 '삼성전자' 키워드로 뉴스 1건을 조회하여 성공 여부 및 상세 로그 출력.

### 11.2. 로깅 및 디버깅 (Testability)
*   `NaverNewsService.ts` 개발 시, 호출 URL 및 응답 데이터를 `KiwoomService`의 `apiLogs`와 유사한 형식으로 관리하거나 콘솔에 상세 출력하여 개발 중 테스트 편의성 확보.
*   네트워크 오류, 키 부적합(401), 한도 초과(429) 등에 대한 명확한 에러 메시지 UI 바인딩.
