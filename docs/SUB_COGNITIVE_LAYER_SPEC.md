# [Sub-Spec] Cognitive Agent Analysis & Hierarchical Reasoning

> [!IMPORTANT]
> 본 문서는 **MAIIS 확장 프로젝트 - 인지 레이어(L3)**의 상세 명세입니다. 마스터 인덱스는 [[00_INVESTMENT_INDEX.md]]를 참조하세요.

본 문서는 MAIIS 시스템의 제3계층인 **인지 및 판단 레이어(Cognitive Layer)**의 구조와 에이전트 간의 계층적 의사결정 체계를 정의합니다. 중앙 집중형 시황 분석 결과를 바탕으로, 대전략을 수립하고 다수의 매매 에이전트가 독립적으로 실행되는 '하향식 의사결정 구조'를 채택합니다.

---

## 1. 의사결정 계층 구조 (Hierarchical Decision Framework)

시스템은 정보의 추상화 수준과 판단의 영향력에 따라 다음과 같이 3단계로 구분됩니다.

### [단계 1] 기초 정보 판독 (Insight Level)
데이터 소스로부터 시장의 객관적 수치와 정성적 이슈를 추출하는 단계입니다.
*   **시장 지표 분석 (Quantitative)**: 금리, 환율, 지수 변동성, 수급 지표 등 수치 기반 판단.
*   **시장 이슈 판독 (Narrative)**: 전쟁, 재해, 정책 변화, 강력한 주도 테마 등 뉴스/텍스트 기반 판단.

### [단계 2] 대전략 수립 (Synthesis Level - Grand Strategy)
단계 1의 정보들을 결합하여 현 시장 현황을 진단하고, 그 원인을 파악하며, 향후 전망을 바탕으로 구체적인 섹터/종목 가이드라인을 수립합니다.
*   **시황 판단**: "현재 시장은 어떠한가?" (지표 분석 결과 통합)
*   **원인 추론**: "왜 이런 현상이 발생하는가?" (이슈 판독 결과 연계 분석)
*   **향후 전망**: "앞으로 시장은 어떻게 될 것인가?" (종합적 예측 및 시나리오 수립)
*   **실행 전략**: "무엇을 사고 무엇을 피해야 하는가?" (유망 섹터/종목 선정 및 기피 대상 확정)
*   **하달 지침**: 전략 에이전트별 자산 배분 및 리스크 한도 설정.

### [단계 3] 매매 전략 집행 (Tactical Level - Distributed Agents)
단계 2에서 하달된 대전략 가이드라인 내에서 개별 알고리즘에 따라 종목을 선정하고 주문을 생성합니다.
*   **매매 에이전트 A (Momentum)**: 주도 섹터 및 급등주 돌파 매매 전담.
*   **매매 에이전트 B (Value/Safety)**: 시장 불안 시 저평가 우량주 및 배당주 중심 포트폴리오 구성.
*   **매매 에이전트 C (Event/Swing)**: 공시 포착, 수급 변동 등 특정 이벤트 기반의 단기 스윙 전략.

## 2. 에이전트별 상세 역할 및 필요 데이터

| 계층 | 에이전트 명칭 | 주요 분석 및 역할 | 필수 데이터 요구사항 |
| :--- | :--- | :--- | :--- |
| **상위** | **Market Insight Monitor** | 시장의 '판'을 읽고 현재 지배 논리 추출 | 매크로 지표, 실시간 뉴스, 유튜브 요약, 공시 빈도 |
| **중간** | **Grand Strategy Master** | 종합 판단 후 에이전트별 스탠스 및 예산 배분 | Insight 결과값, 세계관(World State), 현재 잔고 현황 |
| **하위** | **Trading Agent Suite** | 개별 전략에 따른 종목 발굴 및 타점 선정 | 시황 가이드라인, 종목별 재무/기술적 지표, 호가창 |

---

## 3. 의사결정 프로세스 및 상호관계

1.  **대전략 우선 원칙**: 하위 매매 에이전트는 상위 대전략 에이전트가 발행한 'Market Thesis'와 'Risk Budget'을 절대적으로 준수해야 합니다.
2.  **상호보완적 정보 흐름**: 시황 분석 에이전트가 "전쟁 리스크"를 감지하면, 이를 대전략 에이전트가 수신하여 즉시 모든 매매 에이전트의 예산을 축소하거나 방어형 에이전트의 활성도를 높입니다.
3.  **데이터 독립성**: 각 매매 에이전트는 대전략이라는 공통 분모를 공유하면서도, 종목을 고르는 세부 알고리즘은 상호 독립적으로 작동하여 전략의 다양성을 확보합니다.

---

## 4. 분석 보강 과제 (Future Work)

*   **동적 가중치 모델**: 지표(숫자)와 이슈(내러티브) 중 무엇에 더 가중치를 둘지 결정하는 알고리즘 고도화.
*   **주도 섹터 식별 알고리즘 [Major Challenge]**: 거래대금 쏠림, 글로벌 동조화, 기술적 강도를 결합하여 현재 장세를 지배하는 섹터를 실시간으로 판별하는 로직 개발.
*   **복기 시스템 (Retrospective)**: 장 종료 후 각 에이전트의 판단과 실제 수익률을 비교하여 '대전략'의 정확도를 스스로 튜닝하는 루틴 구축.

## 5. 에이전트별 데이터 요구사항 및 수집 출처

각 계층의 에이전트들이 판단을 내리기 위해 필요로 하는 구체적인 데이터 항목과 그 출처(Source)를 정의합니다.

### 5.1 상위 계층: 기초 정보 판독 데이터 명세

상위 계층 에이전트들은 외부의 원천 데이터를 가장 먼저 접하며, 시스템의 세계관(World State)을 구축하는 데 필요한 기초 데이터를 수집하고 판독합니다.

#### (1) 시장 지표 분석 에이전트 (Quantitative Metrics Agent)
숫자로 표현되는 기술적/거시적 지표를 통해 시장의 물리적 환경을 분석합니다.

| 수집 정보 항목 | 상세 내용 및 용도 | 수집 출처 (Source) |
| :--- | :--- | :--- |
| **국내 지수 시세** | KOSPI, KOSDAQ 현행 지수 및 전일 대비 등락률 | 키움 API / Yahoo Finance |
| **글로벌 지수 시세** | S&P 500, NASDAQ, Dow Jones 지수 (글로벌 동조화 확인) | Yahoo Finance API |
| **원/달러 환율** | USD/KRW 실시간 환율 (외국인 수급 방향성 예측) | Yahoo Finance API |
| **미국 국채 금리** | 미국 10년물/2년물 금리 (유동성 및 할인율 판단) | Yahoo Finance API |
| **한국 국채 금리** | 한국 10년물 국채 수익률 (국내 통화 정책 영향 확인) | Yahoo Finance API |
| **변동성 지수 (VIX)** | CBOE VIX, VKOSPI (시장 공포 및 위험 회피 수준 측정) | Yahoo Finance / 키움 API |
| **시장 에너지 지표** | KOSPI/KOSDAQ 거래대금 합계 및 증감 추이 | 키움 API |
| **수급 주체별 동향** | 외국인, 기관, 개인의 누적 순매수 데이터 | 키움 API |

#### (2) 시장 메인 이슈 판독 에이전트 (Main Narrative Agent)
텍스트 및 사건 기반의 정보를 통해 시장을 움직이는 핵심 논리(Story)를 추출합니다.

| 수집 정보 항목 | 상세 내용 및 용도 | 수집 출처 (Source) |
| :--- | :--- | :--- |
| **실시간 뉴스 헤드라인** | 주요 경제지 및 포털의 실시간 뉴스 (돌발 악재/호재 감지) | Naver News API |
| **섹터별 키워드 빈도** | 뉴스 내 자주 언급되는 단어 (현재 주도 테마 식별) | Naver News / AI 분석 |
| **전문가 시황 브리핑** | 유튜브 채널 텍스트 요약 (고차원적 시장 통찰 확보) | **[계획]** YouTube API |
| **주요 공시 트리거** | 상장사 주요사항보고서, 실적 발표 공시 (이벤트 드리븐) | OpenDART API |
| **글로벌 매크로 이슈** | 미 연준(Fed) 발언, 전쟁, 원자재 폭등 등 외부 변수 텍스트 | Naver/Yahoo/AI 검색 |
| **사회적 감성 점수** | 시장 참여자들의 심리적 낙관/비관 정도 (Sentiment Score) | AI 분석 (L2 레이어 연동) |

### 5.2 중간 계층: 대전략 수립 및 판단 데이터 명세
| 에이전트 | 판단 프로세스 | 필요 데이터 및 요건 |
| :--- | :--- | :--- |
| **Grand Strategy Master** | **현황 진단 (Status)** | 국내외 지수 현황, 업종별 상대 강도(지수 대비 수익률), 환율/금리 추이 |
| | **원인 판독 (Cause)** | 내러티브 요약, 글로벌 섹터 ETF(SOXX 등)와 국내 섹터의 동조화 여부 분석 |
| | **전망 수립 (Future)** | 과거 유사 국면 데이터, 거시 지표 추세, 업종별 수급 집중도 변화량 |
| | **실행 가이드 (Buy/Avoid)** | 거래대금 비중이 폭증한 주도 섹터 선정, 외인/기관 집중 매수 섹터 식별 |
| | **자원 배분 (Budget)** | 실시간 예수금 상태, 주도 섹터 자금 쏠림 정도에 따른 가변적 예산 배정 |

### 5.3 하위 계층: 개별 매매 전략 집행 정보
| 에이전트 | 필요한 정보 구분 | 세부 데이터 항목 | 수집 출처 (Source) |
| :--- | :--- | :--- | :--- |
| **Trading Agent Suite** | **전략별 대전략 지침** | 현재 스탠스(공격/방어), 할당 예산, 집중 투자 섹터 가이드 | 대전략 에이전트 (World State) |
| | **기술적 분석 데이터** | 이동평균선(MA), RSI, 볼린저 밴드, 거래량 패턴 | 키움 API |
| | **재무/본질 분석** | 10개년 재무 스냅샷, 실적 공시, 주요 종목 이벤트 | OpenDART API |
| | **실시간 체결 정보** | 현재가, 호가 잔량, 초단위 체결 강도 | 키움 WebSocket |
