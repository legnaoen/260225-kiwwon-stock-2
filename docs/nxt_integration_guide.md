# 넥스트레이드(NXT) 실시간 시세 연동 계획 가이드

## 1. 개요
한국 대체거래소(ATS)인 넥스트레이드(NXT)의 주식 거래시간 확장(08:00 ~ 20:00, 총 12시간)에 대응하여, 키움증권 REST API 기반 애플리케이션에서 정규장 외 시간에도 실시간 시세 및 체결 데이터를 처리하기 위한 아키텍처 및 구현 계획입니다.

## 2. NXT 거래 시간 세부사항
- **프리마켓**: 08:00 ~ 08:50
- **(거래 중단)**: 08:50 ~ 09:00:30 (시가 산출)
- **메인마켓**: 09:00:30 ~ 15:20
- **(거래 중단)**: 15:20 ~ 15:30 (종가 산출)
- **애프터마켓**: 15:30 ~ 20:00 (체결은 15:40부터)
- **종가매매**: 15:30 ~ 16:00

## 3. 구현 계획 (Phase 1: 데이터 수신)

### 3.1 웹소켓 구독 (WebSocket Registration)
- **현재 상황**: 정규장 주식체결(`type: ['0B']`)과 시장상태(`0s`)만 구독 중.
- **수정 계획**:
  1. 키움증권 최신 API 문서(PDF/XLSX)를 분석하여 **NXT 실시간 주식체결을 의미하는 실시간 타입(Real Type) ID**를 파악합니다.
  2. `electron/websocket.ts`의 `registerItems` 함수에서, 기존 `0B`에 이어 새로운 NXT 체결 타입을 추가 구독 목록에 포함시킵니다.
  ```typescript
  // 예시 코드 
  data: [
      { item: allSymbols, type: ['0B', 'NXT_체결_타입_ID'] },
      { item: [''], type: ['0s'] }
  ]
  ```

### 3.2 수신 라우팅 및 데이터 통합
- 정규장 체결 데이터 송신과 NXT 체결 데이터 송신은 FID 구조가 거의 동일할 것으로 예상됩니다.
- 수신부(`handleMessage`) 로직을 범용화하여, 어떤 타입이든 실시간 체결 데이터가 들어오면 `PriceStore`에 캐싱하고 프론트엔드로 브로드캐스트(`kiwoom:real-time-data`)합니다.

### 3.3 시장 상태 (Market Status) 세분화
- NXT 출범으로 새롭게 추가 혹은 변경된 시장 상태 코드(`215`등)를 매핑합니다.
- `useMarketStore.ts` 내 상태 텍스트 반환 함수를 확장하여 "NXT 프리마켓", "NXT 애프터마켓" 등의 상태가 UI 상태바에 노출되도록 반영합니다.

## 4. 구현 계획 (Phase 2: 주문 라우팅 제어)

### 4.1 자동매매 스케줄링 확장
- 현재 09:00 ~ 15:20(정규장) 위주로 설계된 자동매매 시간을 **사용자의 선택 모델에 따라 08:00 ~ 20:00로 확장**할 수 있도록 설정 UI(자동매매 탭)를 개편합니다.

### 4.2 스마트 오더 라우팅 (SOR) 및 전용 주문 코드 처리
- 거래 시간이 확대됨에 따라 "매도/매수 주문" 호출 시, 키움의 서버가 자동으로 가장 유리한 거래소(KRX vs NXT)로 주문을 보내는지(SOR 기능 적용 유무), 아니면 사용자가 **NXT 전용 주문(예: BuyOrder_NXT, 신규 주문 코드 배정 등)**을 명시해서 호출해야 하는지 문서를 통해 확인하고 `KiwoomService.ts`의 주문 로직을 분기 처리합니다.

## 5. 지침 (개발 착수 전 필수 사항)
- **사전 API 문서 검증**: 어떤 코드 변경보다 먼저 키움 REST API 업데이트 공지와 문서 파일을 분석하여, `NXT 실시간 체결 코드`와 `NXT 전용 주문화면 파라미터`의 정확한 스펙을 확정짓습니다.
- **예외 처리**: 기존 KRX만 바라보도록 하드코딩된 시간 분기(예: 15:20 일괄 시장가 청산 로직 등)가 파괴되지 않도록 상호 연관 관계를 꼼꼼히 체크합니다.
